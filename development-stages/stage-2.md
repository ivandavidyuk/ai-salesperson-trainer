У меня есть работающий Next.js 14 (App Router) бэкенд 
с авторизацией через httpOnly cookie и следующими API:

POST /api/auth/login        — принимает { email, password }
POST /api/auth/logout       — очищает cookie
GET  /api/auth/me           — возвращает { id, email, name }
POST /api/sessions/start    — возвращает { sessionId, wsUrl }
POST /api/sessions/[id]/stop
GET  /api/sessions/[id]/transcript — возвращает [{ role, text, createdAt }]

Создай три страницы:

─────────────────────────────────────
СТРАНИЦА 1: /app/login/page.tsx
─────────────────────────────────────
- Форма: поля email и password, кнопка "Войти"
- При сабмите → POST /api/auth/login
- При успехе → редирект на /session
- При ошибке → показать сообщение "Неверный email или пароль"
- Простой центрированный layout, никаких лишних элементов

─────────────────────────────────────
СТРАНИЦА 2: /app/session/page.tsx
─────────────────────────────────────
Это главный экран. Состояния экрана:

СОСТОЯНИЕ "idle" (до старта):
- Имя пользователя вверху (из GET /api/auth/me)
- Картинка-заглушка по центру
- Кнопка "Начать разговор"
- Кнопка "Выйти" (POST /api/auth/logout → редирект на /login)

СОСТОЯНИЕ "active" (идёт разговор):
- Та же картинка по центру
- Анимация пульсации вокруг картинки (показывает что идёт звонок)
- Кнопка "Пауза"
- Кнопка "Завершить разговор"
- Таймер — показывает сколько длится сессия (формат 00:00)

СОСТОЯНИЕ "paused" (пауза):
- Картинка без анимации
- Кнопка "Продолжить"
- Кнопка "Завершить разговор"
- Таймер остановлен

Логика кнопок:
- "Начать разговор" → POST /api/sessions/start → 
  сохранить sessionId в useState → перейти в состояние active →
  подключиться к WebSocket по wsUrl (пока просто установить 
  соединение, без обработки аудио — это будет следующем этапе)
- "Пауза" → отправить в WebSocket { type: "pause" } → 
  перейти в состояние paused
- "Продолжить" → отправить в WebSocket { type: "resume" } → 
  перейти в состояние active
- "Завершить разговор" → отправить в WebSocket { type: "stop" } →
  POST /api/sessions/{id}/stop → редирект на /transcript/{sessionId}

─────────────────────────────────────
СТРАНИЦА 3: /app/transcript/[id]/page.tsx
─────────────────────────────────────
- GET /api/sessions/[id]/transcript при загрузке страницы
- Список сообщений в хронологическом порядке
- Каждое сообщение:
    role === "user"      → подпись "Менеджер", выравнивание вправо
    role === "assistant" → подпись "Клиент", выравнивание влево
- Время каждой реплики (из createdAt, формат HH:MM:SS)
- Кнопка "Новый разговор" → редирект на /session
- Кнопка "Выйти" → POST /api/auth/logout → редирект на /login

─────────────────────────────────────
ОБЩИЕ ТРЕБОВАНИЯ:
─────────────────────────────────────
- Никаких UI-библиотек (только Tailwind который уже есть в Next.js)
- Минималистичный дизайн
- Все компоненты в /app/components/
- Комментарии на русском языке
- Адаптация под мобильный экран не нужна
