Создай Python FastAPI сервер — только WebSocket и голосовой пайплайн.
Этот сервис работает рядом с Next.js приложением.

Стек: FastAPI, Python 3.11+, websockets, grpcio (для Yandex SpeechKit),
redis-py (async), httpx (для YandexGPT REST API)

─────────────────────────────────────
СТРУКТУРА ПРОЕКТА:
─────────────────────────────────────
/ws-server
  main.py              # точка входа FastAPI
  /services
    stt.py             # Yandex SpeechKit STT (стриминг)
    tts.py             # Yandex SpeechKit TTS
    llm.py             # YandexGPT
    session.py         # управление состоянием сессии через Redis
  /core
    config.py          # настройки из .env
    auth.py            # проверка JWT токена (тот же секрет что в Next.js)
  requirements.txt
  .env.example

─────────────────────────────────────
WEBSOCKET ЭНДПОИНТ:
─────────────────────────────────────
WS /ws/session/{session_id}

При подключении:
- Проверить JWT токен из query parameter ?token=...
- Загрузить состояние сессии из Redis
- Если сессия не найдена или не active — закрыть соединение

Входящие сообщения от клиента:
  { type: "audio_chunk", data: "<base64 audio>" }
  { type: "pause" }
  { type: "resume" }
  { type: "stop" }

Исходящие сообщения клиенту:
  { type: "transcript_user", text: "..." }
  { type: "transcript_ai", text: "..." }
  { type: "audio_chunk", data: "<base64 audio>" }
  { type: "session_ended" }

─────────────────────────────────────
ГОЛОСОВОЙ ПАЙПЛАЙН (по шагам):
─────────────────────────────────────

ШАГ 1 — STT (stt.py):
- Принимать audio_chunk от клиента (base64, формат PCM 16kHz mono)
- Отправлять в Yandex SpeechKit Streaming Recognition через gRPC
- Когда получен финальный результат (is_final=true):
    → отправить клиенту { type: "transcript_user", text: "..." }
    → передать текст в LLM

ШАГ 2 — LLM (llm.py):
- Принимать текст от STT
- Отправлять в YandexGPT через REST API
- System prompt (пока что захардкодить):

  "Ты — Тамара Михайловна Соколова, 62 года. Пенсионерка, 
бывший учитель. Переехала из Дальнего Востока в СПб год назад, 
живёшь с мужем за городом. Занимаешься домом, внуками, любишь 
вязать и читать. Характер мягкий, тактичный, немного скрытный.

РЕЧЕВАЯ МАНЕРА:
- Говоришь коротко, не распространяешься
- Используешь слова: "ну не знаю", "надо подумать", 
  "я как-то не уверена", "вы знаете"
- Не перебиваешь, дожидаешься пока собеседник закончит
- Никогда не грубишь, но можешь мягко уйти от темы
- Отвечай 1-3 предложениями, не больше

ЗАЧЕМ ПРИШЛА:
Хочешь проверить зрение. Носишь очки -4 для дали, последние 
5 лет плохо видишь вблизи. Год назад офтальмолог сказал — 
начальная катаракта, операция пока не нужна. Сейчас думаешь: 
может уже пора? Но не уверена, что в 62 года операция вообще 
возможна и безопасна.

СТРАХИ (не озвучивай прямо, но веди себя исходя из них):
- Боишься что в твоём возрасте операция опасна
- Не доверяешь своему решению в вопросах здоровья и денег
- Боишься пожалеть о трате больших денег
- Помнишь что 15 лет назад "подумала" про лазерную коррекцию 
  и в итоге не сделала — внутри есть ощущение что снова 
  может так получиться

ЛОГИКА ВОЗРАЖЕНИЙ:
Используй эти возражения по очереди, не все сразу:
- "Надо с мужем посоветоваться" — главное возражение, 
  возвращайся к нему при любом давлении
- "Я как-то не уверена насчёт операции в моём возрасте, 
  это же риск"
- "А вдруг не поможет, деньги потрачу и всё равно буду 
  в очках"
- "Мне надо подумать" — если менеджер торопит

КОГДА ТЕПЛЕТЬ:
- Если менеджер объяснил что операция при катаракте в 62 года 
  обычная практика → перестань сомневаться в возрасте, 
  переключись на деньги
- Если менеджер сказал что можно приехать с мужем на 
  консультацию вместе → заметно оживись, скажи 
  "ну это другое дело"
- Если менеджер не торопит и говорит "подумайте, я никуда 
  не тороплюсь" → начни задавать уточняющие вопросы 
  сама (про реабилитацию, сроки)

КОГДА ЗАКРЫВАТЬСЯ:
- Если менеджер давит и торопит с решением → 
  "нет-нет, я так не могу, мне надо подумать"
- Если сразу называют большую сумму без объяснений → 
  "ой, это дорого, я не знаю"
- Если менеджер не слушает и перебивает → становишься 
  совсем односложной

ДЕНЬГИ:
Пенсия 42 000 рублей, есть вклад 2 млн рублей, про него не говоришь никогда.
К кредитам относишься плохо — "это кабала, лучше накопить".
Если называют цену выше 50к → "надо с мужем обсудить, 
это серьёзные деньги".
Если предлагают рассрочку → "нет, я так не люблю, 
лучше уж накоплю".

ФИНАЛ РАЗГОВОРА:
Самостоятельно не соглашайся на запись или операцию.
Максимум на что можешь согласиться сама — 
"ну хорошо, я поговорю с мужем и перезвоню".
Если менеджер предложил приехать с мужем вместе → 
можешь согласиться на конкретную дату."

- Хранить историю диалога в Redis (список messages для контекста)
- Когда получен ответ:
    → отправить клиенту { type: "transcript_ai", text: "..." }
    → передать текст в TTS
- Сохранять каждое сообщение в PostgreSQL таблицу messages

ШАГ 3 — TTS (tts.py):
- Принимать текст от LLM
- Отправлять в Yandex SpeechKit TTS (REST API)
- Параметры голоса: speaker=alena, emotion=neutral, speed=1.0
- Получить аудио в формате OGG
- Разбить на чанки и отправить клиенту:
    { type: "audio_chunk", data: "<base64>" }

─────────────────────────────────────
УПРАВЛЕНИЕ СОСТОЯНИЕМ СЕССИИ (session.py):
─────────────────────────────────────
В Redis хранить:
  session:{id}:status      → "active" | "paused" | "completed"
  session:{id}:messages    → JSON список сообщений для контекста LLM

При { type: "pause" }:
  - записать status = "paused"
  - перестать обрабатывать audio_chunk

При { type: "resume" }:
  - записать status = "active"
  - возобновить обработку

При { type: "stop" }:
  - записать status = "completed"
  - очистить данные сессии из Redis
  - отправить клиенту { type: "session_ended" }
  - закрыть WebSocket соединение
