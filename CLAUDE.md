# Шпаргалка по проекту

Голосовой ИИ-тренажёр по продажам: менеджер разговаривает голосом с ИИ-клиентом
«Тамарой Михайловной», после разговора остаётся расшифровка. Подробности —
в [README.md](README.md), деплой — в [DEPLOY.md](DEPLOY.md).

## Архитектура в двух абзацах

**`frontend/`** (Next.js 14, App Router) — интерфейс, авторизация и вся работа
с БД через Prisma. Здесь же REST API (`app/api/**`). Единственная роль в
голосовом сценарии — выдать одноразовый ws-токен; сам разговор идёт из браузера
по WebSocket напрямую в backend.

**`backend/`** (FastAPI) — голосовой пайплайн STT → LLM → TTS на одном
WebSocket-соединении. Вынесен в отдельный сервис и на зарубежный сервер, потому
что ElevenLabs не отвечает на российские IP. Пишет сообщения в ту же PostgreSQL.

## Команды

```powershell
docker compose up -d                  # PostgreSQL (5433) + Redis (6379)

cd frontend
npm run dev                           # http://localhost:3000
npx tsc --noEmit                      # типы
npm run lint                          # ESLint
npx prisma migrate deploy             # применить миграции
npm run prisma:migrate                # создать миграцию
npm run seed:content                  # советы дня и мотивации
npm run seed:patients                 # пациенты для мастера тренировки
npm run seed:demo                     # демо-аккаунт (после seed:patients)
npm run create-user                   # интерактивно (ввод — за пользователем)

cd backend
.\.venv\Scripts\Activate.ps1
uvicorn main:app --host 127.0.0.1 --port 8000
pytest tests/
```

### Грабли локальной разработки

**Не запускать `npm run build` при работающем `npm run dev`.** Обе команды пишут
в одну папку `.next`: продакшен-сборка перезаписывает её, и dev-сервер теряет
свои чанки. Проявляется как `Cannot find module './276.js'`, 404 на всех
`/_next/static/*` или страница без стилей. Для проверки кода без остановки
сервера есть `npx tsc --noEmit` — он в `.next` не пишет.

**Лечится удалением `.next`** (папка целиком генерируемая, терять нечего):

```powershell
Remove-Item -Recurse -Force frontend\.next
```

Тот же рецепт помогает после переключения веток, если сборка «залипла».

**Prisma на Windows:** `prisma generate` падает с `EPERM`, если запущен
dev-сервер — он держит DLL движка. Остановить `npm run dev`, сгенерировать,
запустить снова.

### Проверка вёрстки в браузере

Если снимаешь стили через `getComputedStyle` в фоновой/скрытой вкладке, помни:
без отрисовки кадров **CSS-переходы не продвигаются**, и замер возвращает
стартовое значение. Из-за этого легко решить, что стиль «не применился», хотя
класс на месте. Перед замером отключай анимации:

```js
document.head.insertAdjacentHTML('beforeend', '<style>*{transition:none !important}</style>');
```

Ещё одна ловушка: `border-[1.5px]` Tailwind **не генерирует** — значение
неоднозначно (ширина или цвет). Нужен префикс: `border-[length:1.5px]`.

## Где что лежит

| Путь | Что |
|---|---|
| `frontend/app/components/` | компоненты дизайн-системы и блоки страниц |
| `frontend/lib/` | серверная и клиентская логика (`auth`, `db`, `home`, `format`, `voiceClient`) |
| `frontend/scripts/` | скрипты: создание пользователя, сиды |
| `frontend/prisma/schema.prisma` | схема БД |
| `backend/main.py` | WS-эндпоинт и оркестрация ходов диалога (`TurnManager`) |
| `backend/services/` | `stt.py`, `tts.py`, `llm.py`, `session.py` |
| `backend/tests/` | pytest |

## Дизайн

Макеты живут в **Claude Design**, проект `ea4fb9e2-0fa6-411f-a86f-60a51be5b26c`
(«Дизайн №2 (ии-тренажер)»). Читать через MCP-инструмент `DesignSync`
(`method: get_file`, файлы вида `Главная.dc.html`). Выбрано направление
**1A «Спокойная клиника»**, дизайн-язык — в `Дизайн-язык.dc.html`.

Внутри `.dc.html` бывает не только вёрстка, но и логика компонента (например,
на главной — ротация «Совет дня»/«Мотивация» и выбор текста по индексу дня).
Её стоит вычитывать, а не додумывать поведение.

**Правила вёрстки:**

- только **светлый десктоп** — мобильную версию и тёмную тему делаем позже,
  разом для всех экранов;
- цвета берём из семантических токенов `frontend/tailwind.config.ts`
  (`brand`, `ink`, `surface`, `line`, `danger`, `good`, `warn`), а не подбираем
  hex в разметке;
- макеты рисуются под конкретную ширину — проверять, что задумано на широких
  экранах, и не растягивать блоки долями там, где размер диктуется содержимым.

## Соглашения по коду

- Комментарии и текст интерфейса — **на русском**. Комментарии объясняют
  «почему», а не пересказывают код.
- Роуты API: `runtime = "nodejs"`, авторизация через `getAuthUser` из
  `@/lib/auth`, доступ к БД через `prisma` из `@/lib/db`.
- Изменяющие роуты проверяют владельца прямо в условии запроса
  (`updateMany({ where: { id, userId } })`), а не отдельным запросом.
- Тестами покрываем чистую логику (например, классификатор поддакиваний);
  голосовой пайплайн проверяется вживую.

## Про продакшен

`main` уезжает в прод автоматически: push → сборка образов → деплой на оба
сервера с рестартом. Мерж = выкатка, рестарт backend обрывает активные
голосовые сессии. Миграции накатываются сами при старте контейнера, данные
(советы дня, пользователи) наливаются скриптами вручную.

Разбора разговоров пока нет: `SessionReview` заполняется только сидом, поэтому
у настоящих разговоров оценок не будет — это ожидаемо, а не баг.
